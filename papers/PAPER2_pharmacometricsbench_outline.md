## Working titles

1. *PharmacometricsBench: a reproducible, tool-grounded benchmark for agentic pharmacometric analysis.*
2. *Measuring whether an AI agent can* run *a pharmacometric analysis, not just describe one.*
3. *A deterministic-truth benchmark and harness for agentic PK/PD, with a proposed tool-fidelity metric.* (Note: tool-fidelity is a design proposal in v0, not an implemented measurement — pick this title only if the paper foregrounds future design.)

## One-sentence thesis

> Agentic pharmacometrics needs a benchmark whose ground truth is the output of *validated deterministic tools*, so that the score measures whether an agent reproduces correct numbers (by calling tools) rather than plausibly hallucinating them — a design we implement, self-test with reference agents, and open for frontier-model evaluation.

## Central contribution (what is genuinely new)

1. A **benchmark + harness** where ground truth is defined as validated compute-tool output on seeded data — self-generating, reproducible, contamination-resistant, requiring no hand-labelling for the deterministic categories.
2. Two **proposed design ideas** distinctive to this domain — **not implemented or measured in v0**, presented as the benchmark's forward design, not as results: a **guided-vs-unguided planning-lift** contrast (quantifying the planning bottleneck for pharmacometrics) and a **tool-fidelity** metric (share of numeric answers that came from a tool call rather than tokens, enabled by an audit trail). The v0 code does not compute either; the oracle-vs-naïve gap is the current, implemented proxy for tool-fidelity.
3. A **reference-agent self-test** (oracle / naïve / keyless mock LLM) that validates the harness before any paid model is run, and a one-line swap to evaluate a real model.

## IMRaD outline (topic sentence + evidence per paragraph)

### 1. Introduction
- **P1 (importance).** LLM agents are being applied to quantitative pharmacology, where a wrong number is not a stylistic error but a safety-relevant one. McCoy and McCoy argue that as agents move the scientist "from executor to orchestrator," the central risk becomes evaluating recommendations that are *difficult to verify independently*, and that the field has not yet built the means to do so (McCoy & McCoy, *Clin. Pharmacol. Ther.* 2026, doi:10.1002/cpt.70380). A benchmark whose ground truth is the output of validated deterministic tools is one such means: it makes "did the agent compute the right number" checkable rather than a matter of trust.
- **P2 (what is known).** General agent benchmarks exist; the closest, an early-drug-discovery agent benchmark, shows frontier agents plateau (~half of tasks) and that *unguided planning* is the main bottleneck (most tasks become solvable when expert step sequences are provided). `[CITATION NEEDED: Scale DrugDiscoveryBench — 82 tasks; ~50% ceiling; 76/82 solvable when guided]`
- **P3 (gap).** No benchmark evaluates whether an agent can *execute* a pharmacometric analysis (NCA, BE, dose-proportionality, structural PK) with correct, reproducible numbers; scoring an LLM's free-text answer against a fixed key does not capture whether it used validated computation.
- **P4 (aim).** We introduce PharmacometricsBench: deterministic tool-grounded tasks, tolerance-based grading, and reference agents that self-test the harness; we release the design and invite frontier-model evaluation.

### 2. Methods
- **Task model.** Each task = prompt + dataset + typed targets (each a value + tolerance rule: relative, absolute, twofold/GMFE band, or exact). Fully reproducible from a seed.
- **Ground truth = validated tool output.** For deterministic categories the answer key is generated by running the analysis tool on the seeded data, so tasks are generatable, not hand-labelled. The tools carry their own reference-validation suite (`tests/reference/test_theophylline.py`, 8 tests), which checks them against reference values; the published source for those reference values still needs citing. `[CITATION NEEDED: Theophylline reference values]`
- **v0 categories (5).** NCA; average bioequivalence; dose proportionality (power model); one-compartment structural PK parameter recovery; and steady-state **exposure prediction** by forward simulation (predict Cmax_ss and AUC_tau for a multiple-dose regimen), graded within **2-fold** — the accuracy criterion used for human-PK-prediction method comparisons (e.g. Käser et al., *Mol. Pharm.* 2026). State the deferred categories (PBPK, identifiability, TDM) and why (need fixtures or literature keys).
- **Grading & scoring.** Per-target pass/fail under tolerance; task score = mean over targets; overall = mean over categories (equal weight so a large easy category cannot dominate).
- **Reference agents.** Oracle (calls the tools); naïve (tool-free, plausible-but-wrong); keyless mock LLM (prompt → text → parse pipeline). The oracle validates that tasks are well-formed; the naïve/mock quantify the tool-free floor.
- **Distinctive metrics (design).** Guided-vs-unguided planning-lift; tool-fidelity rate (auditable). Present as the benchmark's design contributions; note which are implemented vs specified.
- **Motivating comparator.** Contrast scope with the early-discovery agent benchmark (that one stops at discovery; ours starts at molecule + data). `[CITATION NEEDED]`

### 3. Results
- **R1 — harness is well-formed.** The tool-calling oracle scores **1.000** overall and in every category — confirming the tasks are correctly specified.
- **R2 — the benchmark discriminates.** A tool-free naïve agent scores **0.778** overall (per-category BE 0.67, structural PK 0.50, dose-prop 0.92, exposure 0.92, NCA 0.89) — a large, honest gap from the oracle.
- **R3 — the LLM pipeline is faithful.** A keyless mock LLM routed through the full prompt→parse pipeline reproduces the naïve floor exactly (**0.778**), demonstrating the evaluation plumbing is correct independent of any paid model. *(This is a plumbing result, not a claim about any real model's ability.)*
- **R4 — reproducibility.** The task set is seed-deterministic; the harness is one command, and the runner reports each agent's errored/parse-gap counts so a crash is never mistaken for a wrong answer (the reference agents show 0 errored, 0 parse-gap — the 0.778 is genuinely wrong answers). 18 regression tests pass across the two bench suites (12 in `test_pharmacometricsbench.py` + 6 in `test_pmbench_llm.py`). Captured run pinned in `papers/PAPER2_bench_run.txt`.

### 4. Discussion
- **Interpretation.** Tool-grounded truth reframes the question from "did the model output the right string" to "did the agent produce correct, reproducible numbers via validated computation" — the relevant question for regulated pharmacometrics.
- **Comparison to prior work.** Position against the early-discovery agent benchmark and general LLM-eval practice. `[CITATION NEEDED]`
- **Answering the call for verifiable agentic pharmacology.** McCoy and McCoy (2026) hold that agentic AI helps only where its recommendations can be evaluated, and warn of the failure modes that follow delegating scientific judgment. A tool-grounded benchmark operationalises that evaluation for the execution layer: the score measures whether an agent reproduced a validated computation, and the tool-fidelity metric (proposed) would further separate a right number obtained by calling a tool from one produced by tokens. We are explicit that this addresses verifiability of *execution*, not the deeper drivers of clinical failure that review foregrounds, and that v0 reports reference agents rather than a frontier model.
- **Limitations (honest, prominent).** (i) **No real frontier-model results yet** — reference agents only; the naïve = mock-LLM equivalence proves the pipeline, not any model's competence. (ii) v0 is five deterministic categories; the scientifically hardest (PBPK first-in-human prediction, identifiability, TDM) are deferred. (iii) 30 tasks — small; scaling planned. (iv) Ground truth = the app's own tools, so the benchmark measures agreement with those (independently validated) tools, not a per-task re-derivation. (v) The exposure category grades within 2-fold — deliberately generous (the field standard), so it is discriminating between correct forward simulation and crude closed-form guesses but not a fine-grained accuracy test.
- **Implications / future work.** Evaluate frontier models (one-line client swap); add PBPK/identifiability categories from validated published models (the domain moat); run the guided-vs-unguided and tool-fidelity analyses at scale.
- **Moat category (designed, not implemented — needs curated data).** A **first-in-human PK prediction** category, modelled on Käser et al. (*Mol. Pharm.* 2026): given a drug's preclinical/in-vitro inputs, predict human AUC∞ and Cmax, graded within 2-fold against observed clinical values, and compared across methods (standard PBPK / high-throughput PBPK / ML / ensemble). This requires either a PBPK engine or a **curated set of drugs with published human PK** (observed AUC/Cmax) as answer keys — data the author must supply; it must not be fabricated. This is the category that would make the benchmark a genuine analogue of the Käser method comparison and of the cross-engine ensemble result (companion paper).

### 5. Conclusion
- Restate: a deterministic, tool-grounded benchmark makes "can an agent *run* pharmacometrics" measurable and reproducible; the harness is validated by reference agents and ready for frontier-model evaluation, which is the immediate next step.
