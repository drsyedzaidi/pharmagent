Dealing with Messy Data in R

Phase 03: Joining & Reshaping Multi-Table Data

This phase moves from single-table cleaning to combining data that lives in more
than one place — the normal state of real studies, where subjects, dosing events,
and lab draws are never in the same spreadsheet. You will learn the five core
dplyr join verbs, learn to catch the single most common real-world join bug
(silent row multiplication from a non-unique key), and assemble a realistic
three-table popPK dataset (dosing, observations, covariates) into one
event-ordered table. One continuity set of three related tables — subjects,
dosing, and labs — runs through all three lessons with deliberate variations
(a duplicate-key labs table in Lesson 02) so each new join concept lands on
data you already recognize.

Learning Objectives

  • Use left_join(), inner_join(), full_join(), anti_join(), and semi_join()
    correctly, and predict exactly which rows survive and which columns get
    NA-filled for each
  • Join on multiple keys and control matching explicitly with by =
  • Recognize silent row multiplication caused by a non-unique join key, and explain
    why it is the most common and most dangerous join bug
  • Validate a join before trusting it: check nrow() before/after, check
    anyDuplicated() on each side's key, and use dplyr's relationship = argument
    to fail loudly instead of multiplying silently
  • Assemble a realistic popPK-style dataset from separate dosing, observations, and
    covariates tables into one ID+TIME-ordered table, with validation that no
    covariate row was duplicated or dropped in the process

Prerequisites

Phase 02 (Data Transformation with dplyr & tidyr) — the five dplyr verbs, grouped
operations, and tidyr reshaping. install.packages(c("tidyverse")) (uses
dplyr for joins; no additional packages required).
