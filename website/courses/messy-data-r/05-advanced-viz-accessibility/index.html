Dealing with Messy Data in R

Phase 05: Advanced Visualization & Accessibility

This phase moves past default ggplot2 output and treats colour as a design decision
with consequences. You will learn to match colour scales to data type (sequential,
diverging, qualitative), understand why the default hue wheel actively misleads a
meaningful fraction of readers with colour-vision deficiency, adopt perceptually
uniform and colour-blind-safe palettes (viridis, Okabe-Ito, colorspace) as defaults,
and build a reusable custom theme with correct ggsave() export settings for slides,
web, and print journals. By the end you can audit any figure for colour-blind safety
and ship it at publication resolution in the right file format.

Learning Objectives

  • Choose sequential, diverging, or qualitative colour scales based on the underlying data type, and explain why rainbow/jet colormaps distort perception
  • Identify why default ggplot2 hues are not colour-blind-safe, and apply viridis, colorspace, and Okabe-Ito palettes as safe alternatives
  • Simulate colour-vision deficiency to audit a plot, and apply the redundant-channel rule (shape/linetype/direct labels) so no critical distinction depends on colour alone
  • Build a reusable custom ggplot2 theme and export publication-ready figures with ggsave() at correct dpi, units, and file format for the target medium

Prerequisites

building a plot layer by layer. Packages: tidyverse, viridis, colorspace,
colorblindr (optional; install from GitHub — see Lesson 02), scales.
