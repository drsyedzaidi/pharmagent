Dealing with Messy Data in R

Phase 06: Publication Figures — Multi-Panel & Case Study

This is the capstone phase of the course. Every prior phase built one skill in
isolation: import (01), clean with dplyr/tidyr (02), join multi-table data (03), the
ggplot2 grammar of graphics (04), and accessible colour/theme design (05). Phase 06 does
not teach a new data skill — it teaches composition: taking several individually
correct ggplot2 panels and assembling them into one tagged, aligned, publication-grade
figure, then exporting it at the resolution and format a journal actually requires. The
final lesson runs the entire pipeline end to end, from raw messy data to a submission-
ready PDF/PNG, integrating everything from Phases 01–05 into one deliverable.

Learning Objectives

  • Compose multi-panel figures with patchwork using +, /, |, and nested
    parentheses for arbitrary layouts
  • Control panel geometry with plot_layout(ncol=, nrow=, widths=, heights=) and add
    panel tags (A, B, C…) with plot_annotation(tag_levels=)
  • Collect a shared legend across panels with plot_layout(guides = "collect")
  • Use cowplot::plot_grid() as an alternative composition engine, with align/axis
    for pixel-perfect axis alignment across heterogeneous panel types
  • Extract and manually place a legend with cowplot::get_legend(), and annotate a
    composed figure with ggdraw() + draw_label()/draw_image()
  • Know when to reach for patchwork (fast, tag-friendly, general layout) versus
    cowplot (axis alignment across dissimilar plot types, manual legend control)
  • Run the full pipeline: import messy popPK data → clean → join covariates → build 3–4
    ggplot2 panels → apply an accessible style guide → compose → export at 300+ dpi PNG
    and vector PDF

grammar, accessible colour/theme design). install.packages(c("tidyverse", "patchwork", "cowplot", "scales", "viridis")).
