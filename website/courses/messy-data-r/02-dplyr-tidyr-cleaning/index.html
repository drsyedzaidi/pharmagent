Dealing with Messy Data in R

Phase 02: Data Transformation with dplyr & tidyr

This phase turns raw, imported-but-still-messy tables into clean, analysis-ready
tibbles. You will learn the five core dplyr verbs as composable building blocks,
master grouped aggregation and group-wise transforms (including the window-function
patterns every longitudinal or repeated-measures dataset needs), and reshape data
between wide and long forms with tidyr. One continuity dataset — a simulated messy
clinical/PK trial extract with inconsistent sex coding, multiple lab-value columns, a
compound subject-visit ID, and repeated measures per subject — runs through all three
lessons, so each new verb is applied to the same recognizable mess instead of a fresh
toy example every time.

Learning Objectives

  • Compose select(), filter(), mutate(), arrange(), and summarise() into
    pipeable, non-mutating data transformation chains
  • Recode messy categorical values safely with case_when(), and recognize how
    filter() silently drops NA rows unless you test for them explicitly
  • Use group_by() + summarise() for aggregation, and understand the .groups=
    argument and the "list column"/forgotten-ungroup() traps
  • Distinguish mutate()-after-group_by() (group-wise transform, same row count) from
    summarise() (collapses to one row per group), and apply window functions (lag(),
    lead(), cumsum(), row_number()) inside grouped mutate()
  • Reshape data with pivot_longer() and pivot_wider(), and split/combine compound
    columns with separate_wider_delim() and unite()
  • Explain why reshaping is a means to tidy data, not a fixed destination — wide and
    long can both be tidy depending on the unit of observation
