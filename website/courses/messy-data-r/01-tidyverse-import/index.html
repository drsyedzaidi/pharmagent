Dealing with Messy Data in R

Phase 01: Tidyverse Foundations & Data Import

This phase builds the ground floor every messy-data workflow stands on: what the
tidyverse actually is, why tibbles behave differently from base data frames, how to
import real-world CSV/Excel files without silently corrupting them, and a systematic
audit you run on any new dataset before touching a single dplyr verb. By the end you
can import a genuinely messy pharmacometric dataset and know exactly what is wrong
with it before you try to fix it.

Learning Objectives

  • Explain the tidyverse's shared design philosophy and the tibble data structure, and contrast tibbles with base data.frame
  • Use the pipe (|> and %>%) to compose left-to-right data pipelines
  • State Wickham's tidy-data definition and recognize wide-vs-long violations
  • Import CSV, delimited, and Excel files with readr/readxl, using explicit col_types and problems() to catch silent parsing failures
  • Recognize the common import failure modes: encoding, delimiters, thousands separators, date formats, whitespace, mixed-type columns, Excel serial dates
  • Run a systematic pre-cleaning audit (glimpse(), class checks, duplicate detection, categorical-level checks, sentinel missing codes) before any transformation

Prerequisites

Base R (vectors, data frames, functions, basic control flow). No tidyverse experience
required. install.packages(c("tidyverse", "readxl", "skimr", "janitor")).
