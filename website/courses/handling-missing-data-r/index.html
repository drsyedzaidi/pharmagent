Handling Missing Data in R — From General Theory to Pharmacometric Practice

A hands-on course on diagnosing and handling missing data in R. It starts from the
statistical foundations every analyst needs — the MCAR/MAR/MNAR taxonomy, diagnostics,
deletion, single and multiple imputation — and then carries those ideas into the setting
where they bite hardest: population pharmacokinetic (popPK) and PK/PD analysis, where
missingness shows up as absent dose/time records, below-quantification-limit (BLQ)
concentrations, and incomplete covariates. Each lesson derives the method from first
principles, implements it in R, and ships a keepable artifact — a checklist, decision
tree, or analysis script — for use on real projects.

This course was built by fusing the syllabus of the Physalia Handling Missing Data in R
workshop with the pharmacometric missing-data literature (Beal 2001 on BLQ; Johansson &
Karlsson 2013 on missing covariates; Jonsson & Nyberg 2024 on FREM).

Who This Is For

Pharmacometricians, clinical pharmacologists, and quantitative analysts who already use R
and want a single course that covers both the general missing-data toolkit (naniar,
VIM, mice) and its pharmacometric specializations (BLQ likelihood methods, covariate
imputation, FREM). Phases 01–04 stand on their own for any analyst; phases 05–06 assume
familiarity with population modeling.

Prerequisites

  • R basics: dplyr, ggplot2, reading and reshaping data frames
  • Introductory statistics: means, variances, regression, likelihood
  • For phases 05–06: population PK concepts (NONMEM/nlmixr2 dataset structure, IIV, covariates)
  • Packages: install.packages(c("naniar","VIM","visdat","mice","ggmice","missForest","softImpute","dplyr","ggplot2"))
  • For phases 05–06 (optional but recommended): mrgsolve, nlmixr2

What You Will Learn

  • Distinguish MCAR, MAR, and MNAR — and why the mechanism dictates which methods are valid
  • Explore and visualize missingness in R with shadow matrices, VIM, and naniar
  • Apply deletion, single imputation, and stochastic regression imputation — and see how each biases estimates
  • Run multiple imputation by chained equations (mice) and pool results with Rubin's rules
  • Use iterative / machine-learning imputation (EM, SoftImpute, missForest)
  • Handle the missing-data problems specific to popPK datasets: absent time/dose records, dose imputation with ADDL/SS
  • Choose and implement a BLQ method (Beal's M1–M7), including the censored-likelihood M3 method in NONMEM and nlmixr2
  • Impute time-varying covariates within subjects (LOCF/NOCB) and handle wholly missing covariates (CC, SI, MI, full ML)
  • Build a Full Random Effects Model (FREM) that handles missing covariates implicitly

Key References

  • Rubin DB (1976). Inference and missing data. Biometrika 63:581–592.
  • Little RJA, Rubin DB (2019). Statistical Analysis with Missing Data, 3rd ed. Wiley.
  • van Buuren S (2018). Flexible Imputation of Missing Data, 2nd ed. CRC Press. (free online: https://stefvanbuuren.name/fimd/)
  • Beal SL (2001). Ways to fit a PK model with some data below the quantification limit. J Pharmacokinet Pharmacodyn 28(5):481–504.
  • Bergstrand M, Karlsson MO (2009). Handling data below the limit of quantification in mixed effect models. AAPS J 11(2):371–380.
  • Ahn JE et al. (2008). Likelihood based approaches to handling data below the quantification limit using NONMEM VI. J Pharmacokinet Pharmacodyn 35:401–421.
  • Johansson ÅM, Karlsson MO (2013). Comparison of methods for handling missing covariate data. AAPS J 15(4):1232–1241.
  • Jonsson EN, Nyberg J (2024). Full random effects models (FREM): A practical usage guide. CPT Pharmacometrics Syst Pharmacol 13:1297–1308.
  • Bauer RJ (2019). NONMEM tutorial part II: estimation methods and advanced examples. CPT Pharmacometrics Syst Pharmacol 8(8):538–556.

Tooling

  • R (>= 4.2)
  • General missing-data: naniar, VIM, visdat, mice, ggmice, missForest, softImpute
  • Pharmacometric: mrgsolve (simulation), nlmixr2 (fitting), NONMEM syntax shown where relevant
  • dplyr, ggplot2, rmarkdown

Pedagogy: Learn → Build → Ship → Check

Every lesson runs the same four-move loop. You derive the concept, implement it in R,
walk away with a reusable artifact, and answer 3 check questions before advancing.
