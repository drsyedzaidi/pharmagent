Handling Missing Data in R

Phase 05: Missing Data in Population PK — Structure, Time, Dose, and BLQ

Phases 01–04 built the general missing-data toolkit on rectangular, one-row-per-subject
data. Pharmacometric data are not rectangular. A population PK (popPK) dataset is a
time-ordered stream of event records — each row is either a dose or an observation —
and missingness in that stream behaves very differently depending on which item is
missing and why. This phase carries the MCAR/MAR/MNAR taxonomy from Phase 01 into the
popPK event dataset, then works through the three missingness problems every analyst
meets: absent date/time and dose records, and — the hard one — concentrations below the
limit of quantification (BLQ), which are MNAR by construction and are handled with
Beal's censored-likelihood methods M1–M7.

By the end you can look at a raw NONMEM/nlmixr2 dataset, classify every kind of
missingness it contains, impute a dose history correctly with ADDL/II/SS, choose a
BLQ method defensibly from the %BLQ, and implement the M3 censored likelihood in both
NONMEM $ERROR and nlmixr2.

Learning Objectives

  • Read a popPK dataset as an event stream and explain why a missing dose record damages more observations than a missing concentration (dosing propagates forward; observations do not)
  • Map each classical missing-data type (MCAR/MAR/MNAR) onto the standard NONMEM data items (ID TIME DV AMT CMT EVID MDV ADDL II SS BLQ + covariates)
  • Handle missing date/time and missing dose amount by exclusion, and impute a between-visit dose history using the two encodings ADDL/II and SS/II
  • Explain why BLQ data are MNAR and why a BLQ record still carries information (C < LLOQ)
  • Enumerate Beal's methods M1–M7 precisely, state the bias each introduces, and pick a method from the %BLQ using the modern (Bergstrand & Karlsson, Ahn) consensus
  • Implement the M3 censored likelihood: the annotated NONMEM $ERROR` block (F_FLAG / `PHI()` / LAPLACIAN / `$SIGMA 1 FIX) and the nlmixr2 cens/limit analogue

Prerequisites

  • Phase 01 (MCAR/MAR/MNAR taxonomy) — the mechanism vocabulary is used throughout
  • Population PK dataset structure: the NONMEM data items, EVID, one-compartment vs two-compartment models, IIV and residual error
  • R basics: dplyr, ggplot2. For the BLQ simulation, mrgsolve is used where available; a closed-form fallback is provided
  • Optional: nlmixr2 for the M3 fitting analogue
