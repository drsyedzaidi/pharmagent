Non-negotiables

Four guarantees, enforced by the system — not by trust.

Each principle is a hard constraint in the architecture, verifiable in the code and the audit trail.

01 / privacy

Schema-only

Raw patient data never reaches the model. A SchemaExtractor passes metadata summaries only — the LLM sees structure, not subjects.

02 / determinism

Tools, not tokens

Every number comes from deterministic Python — numpy, scipy, pandas. The model reasons about strategy; it never computes the result.

03 / oversight

Human in the loop

Review gates sit at every scientific decision point. Nothing advances past a checkpoint without an explicit approve.

04 / traceability

Hash-chain audit

Every tool call is wrapped in an append-only SHA-256 chain. The full analytical path is reconstructable, end to end.

Why it exists

General-purpose AI can describe a PK analysis. It can't be trusted to run one.

The problem with chatbot pharmacometrics

A raw LLM guesses numbers, forgets its own steps, and leaks whatever you paste in. For regulated science, that fails on day one.

  • Fabricated or non-reproducible parameter values
  • Patient-level data sent to a third-party model
  • No record of how a result was reached
  • No place for a modeller to intervene

PharmAgent's answer

Split the roles. Let agents plan the analysis; let validated tools carry it out; keep a person in command of the science.

  • Reproducibility comes from the tools, not the model
  • Metadata-only summaries protect the dataset
  • Every step recorded in a tamper-evident chain
  • Approve or reject at each decision gate

How it works

Every request follows the same auditable spine.

A user prompt is routed, executed by tools, written to shared state, and held at a review gate — with the whole path recorded as it goes.

step 1

User

Asks in plain language — “run NCA on this dataset.”

step 2

Supervisor

Scores intent, routes to the right domain agent.

step 3

Agent → tools

Calls deterministic tools; all math runs in Python.

step 4

PharmState

Results written to typed shared state, access-controlled.

step 5

Review gate

You approve or reject before the next step runs.

Compliance posture

Designed toward 21 CFR Part 11 from the first commit.

Traceability is not correctness — a human still verifies the science. But the record is built for regulated environments by construction.

Tamper-evident audit export

The full SHA-256 hash chain exports as an append-only record of every tool call and decision.

Read-only raw data

Source datasets stay read-only; the model only ever receives schema-level metadata.

E-signature metadata

Review-gate approvals capture the actor and reason, ready for electronic-signature workflows.

Per-tool validation

Each computational tool carries its own validation documentation and deterministic tests.

Reproducibility record

Runs capture versions, dataset hashes and git provenance so any result can be reconstructed.

Human-in-the-loop by design

No scientific decision advances without an explicit human approval at the gate.
